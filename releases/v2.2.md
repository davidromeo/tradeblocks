# TradeBlocks MCP Server 2.2 Beta — Massive.com Integration + Trade Replay + Greeks

You can now import market data directly from Massive.com, replay any historical trade with minute-level greeks, and test exit policies across your entire portfolio. The VIX data model has been normalized to support any tenor — VIX, VIX9D, VIX3M, or whatever comes next. No more manual CSV exports for market data if you have a Massive subscription.

This is a beta — looking for feedback before merging to stable.

## Massive.com API Integration

A new `import_from_massive` tool replaces the manual CSV export workflow for market data. Point it at a ticker and date range, and it fetches daily OHLCV, intraday bars (1m/5m/15m/1h), or option minute bars directly from Massive's REST API. Zod-validated responses, automatic pagination, and ticker normalization (SPX, I:VIX, O:SPX... formats handled transparently).

For VIX context, `target_table: "context"` is a convenience shorthand that fetches VIX + VIX9D + VIX3M in one call and triggers enrichment automatically. Set `MASSIVE_API_KEY` in your environment — everything else works exactly as before without it.

## Trade Replay

The `replay_trade` tool fetches historical option minute bars from Massive, resolves your tradelog leg descriptions to OCC tickers (e.g., `SPY 450P/445P` → `O:SPY250117P00450000`), and combines them into a minute-by-minute strategy P&L path with MFE and MAE. Two ways to use it: pass explicit legs, or point it at a block_id + trade_index from your existing tradelogs.

Option bars are cached in `market.intraday` after the first fetch — replaying the same trade again is instant, no API call needed.

## Greeks Engine

Every replay point now includes per-leg greeks (delta, gamma, theta, vega, IV) and net position greeks. Black-Scholes with Newton-Raphson IV solver handles regular options. For 0DTE options where BS gamma explodes, a Bachelier normal model kicks in automatically below 0.1 days to expiry. Every greeks result includes a `model` field (`'bs'` or `'bachelier'`) so you know which one computed it.

## Exit Trigger Analysis

Three new tools for understanding when and why to exit trades:

- `analyze_exit_triggers` — evaluates 14 trigger types against a single trade's replay path. Supports profitTarget, stopLoss, trailingStop, DTE/DIT exits, clock time, underlying moves, position delta, per-leg delta, VIX moves, VIX9D/VIX ratio, and stop-loss ratio triggers. Shows which trigger fires first and compares against your actual exit.
- `decompose_greeks` — breaks down P&L into delta, gamma, theta, vega, and residual contributions between any two points. For calendar strategies, includes per-leg-group vega attribution showing front vs back month IV divergence.
- `batch_exit_analysis` — the portfolio-level version. Replays matching trades from a block, evaluates a candidate exit policy against each one, and returns aggregate stats (win rate, Sharpe, profit factor, max drawdown) directly comparable to `get_statistics` output. Per-trigger attribution tells you which trigger is doing the heavy lifting.

Percentage-based thresholds (`unit: 'percent'`) are supported for profit targets and stop losses. Leg groups let you set different triggers for each structure in a multi-leg position.

## Live Options Snapshot

`get_option_snapshot` fetches real-time option chain data (greeks, IV, open interest, quotes) from Massive's snapshot endpoint. Auto-paginates through the full chain. When Massive returns empty greeks for a contract, Black-Scholes fills them in as a fallback — each contract shows `greeks_source: 'massive'` or `'computed'` so you know which is which.

## Normalized VIX Term Structure

VIX data has been restructured. Instead of hardcoded VIX_Close, VIX9D_Close, VIX3M_Close columns in `market.context`, VIX tenors are now regular ticker rows in `market.daily` with `ivr` and `ivp` columns. Cross-ticker derived fields (Vol_Regime, Term_Structure_State, Trend_Direction) live in a new `market._context_derived` table.

The enrichment pipeline discovers VIX tickers dynamically — add VIX1D or VIX6M and it computes IVR/IVP automatically. All query builders, tool descriptions, and example queries have been updated. `describe_database` now returns available VIX tenors with ready-to-use query patterns.

Existing databases migrate automatically on first startup. `market.context` is preserved for backward compatibility.

## SQL Confirm Gate

`run_sql` now requires `confirm: true` for all mutating operations (DELETE, UPDATE, INSERT, CREATE, ALTER, DROP, TRUNCATE). Without it, DELETE and UPDATE return a preview of how many rows would be affected. Prevents accidental data loss when exploring via SQL.

## New Tools

| Tool | What it does |
|------|-------------|
| `import_from_massive` | Import daily, intraday, or option bars from Massive.com API |
| `replay_trade` | Minute-level P&L path with MFE/MAE and per-leg greeks |
| `get_option_snapshot` | Live option chain with greeks, IV, and open interest |
| `analyze_exit_triggers` | Evaluate 14 exit trigger types against a trade's replay |
| `decompose_greeks` | P&L decomposition into greek factor contributions |
| `batch_exit_analysis` | Test exit policies across entire blocks with aggregate stats |

## Breaking Changes

None. All existing tools maintain their behavior. The VIX schema migration is automatic and additive.
