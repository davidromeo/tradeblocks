# TradeBlocks MCP Server 2.3 — Greeks Attribution + Provider Infrastructure + Bug Fixes

Greeks attribution across your portfolio, a pluggable provider layer with full ThetaData support, and targeted quote enrichment. Plus critical replay fixes for calendar spreads.

**Greeks Attribution** — `get_greeks_attribution` decomposes P&L into delta, gamma, theta, vega (optionally charm, vanna). Summary mode shows what drives a strategy across all trades. Instance mode gives per-day Greek time-series for a single trade. Execution edge attribution quantifies the gap between mark-to-market and actual P&L.

**Provider Abstraction** — `MarketDataProvider` interface with capabilities declaration, contract lists, and bulk downloads. Switch providers via `MARKET_DATA_PROVIDER` env var. Massive.com remains the default.

**ThetaData Provider** — Full ThetaData Terminal integration. Daily and intraday bars, option snapshots with greeks, contract list reference, auto-start for the Java process. Monthly chunking, retry with backoff, graceful NO_DATA handling.

**Quote Enrichment** — `enrich_quotes` for targeted NBBO enrichment of specific option tickers. Density-based skip logic, provider-agnostic via `fetchQuotes()` interface. `mergeQuoteBars` creates synthetic minute bars from bid/ask data for illiquid contracts.

**Bug Fixes** — Calendar spread replay now fetches bars from trade open (not near-term expiry). Trade indexing is deterministic via `rowid` tiebreaker. Sparse cached option data no longer rejected by the partial-cache heuristic. Dense trade-only caches correctly trigger quote backfill.

## What's New

### get_greeks_attribution

```
get_greeks_attribution({
  block_id: "main-port",
  strategy: "War Time Put calendar",
  mode: "summary",        // or "instance" with trade_index
  skip_quotes: true,       // fast mode, uses cached bars
  detailed: true           // adds charm, vanna
})
```

Summary mode returns factor percentages across all trades — reveals whether a strategy profits from theta, vega, or delta. Instance mode returns per-step Greek P&L for a single trade.

### decompose_greeks improvements

- Calendar spread date range fix — bars now fetched from trade open through latest expiry
- Stable trade ordering — `rowid` tiebreaker prevents index drift
- Execution edge = actual P&L minus mark-to-market P&L

### Provider infrastructure

Set `MARKET_DATA_PROVIDER` in `.mcp.json` env to select your provider. Both normalize to the same `BarRow` and `OptionContract` types.

**Massive.com** (default):
```json
{
  "mcpServers": {
    "tradeblocks": {
      "env": {
        "MARKET_DATA_PROVIDER": "massive",
        "MASSIVE_API_KEY": "your-key",
        "MASSIVE_DATA_TIER": "ohlc | trades | quotes"
      }
    }
  }
}
```

`MASSIVE_DATA_TIER` controls which endpoints are available:
- `ohlc` — trade-based OHLCV minute bars only (basic tier, all providers)
- `trades` — tick-level trades available (mid tier)
- `quotes` — NBBO bid/ask quotes available, enables synthetic minute bars for illiquid contracts (top tier)

**ThetaData:**
```json
{
  "mcpServers": {
    "tradeblocks": {
      "env": {
        "MARKET_DATA_PROVIDER": "thetadata",
        "THETADATA_HOME": "/path/to/Thetadata"
      }
    }
  }
}
```
