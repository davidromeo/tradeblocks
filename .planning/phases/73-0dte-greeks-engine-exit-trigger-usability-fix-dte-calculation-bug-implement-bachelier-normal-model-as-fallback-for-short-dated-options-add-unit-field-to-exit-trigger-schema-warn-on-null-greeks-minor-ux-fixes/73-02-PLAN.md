    TDD GREEN phase -- implement in exit-triggers.ts:

    **Step 1: Add `unit` and `entryCost` to ExitTriggerConfig** (per D-07):
    Add two fields to the interface:
    ```typescript
    export interface ExitTriggerConfig {
      type: TriggerType;
      threshold: number;
      unit?: 'percent' | 'dollar';  // D-07: default 'dollar', backwards compatible
      // ... existing optional fields ...
      // Internal: set by handler, not by user
      entryCost?: number;  // D-11: used when unit='percent' to compute dollar threshold
    }
    ```

    **Step 2: Modify profitTarget case** in `evaluateTrigger` (per D-08, D-09):
    ```typescript
    case 'profitTarget': {
      let dollarThreshold = threshold;
      if (trigger.unit === 'percent' && trigger.entryCost != null) {
        dollarThreshold = threshold * Math.abs(trigger.entryCost);
      }
      if (pnl >= dollarThreshold) {
        fired = true;
        detail = trigger.unit === 'percent'
          ? `P&L $${pnl.toFixed(2)} >= ${(threshold * 100).toFixed(0)}% of $${Math.abs(trigger.entryCost!).toFixed(2)} ($${dollarThreshold.toFixed(2)})`
          : `P&L $${pnl.toFixed(2)} >= target $${dollarThreshold.toFixed(2)}`;
      }
      break;
    }
    ```

    **Step 3: Modify stopLoss case** similarly:
    ```typescript
    case 'stopLoss': {
      let dollarThreshold = threshold;
      if (trigger.unit === 'percent' && trigger.entryCost != null) {
        dollarThreshold = threshold * Math.abs(trigger.entryCost);
      }
      if (pnl <= -dollarThreshold) {
        fired = true;
        detail = trigger.unit === 'percent'
          ? `P&L $${pnl.toFixed(2)} <= -${(threshold * 100).toFixed(0)}% of $${Math.abs(trigger.entryCost!).toFixed(2)} (-$${dollarThreshold.toFixed(2)})`
          : `P&L $${pnl.toFixed(2)} <= stop -$${dollarThreshold.toFixed(2)}`;
      }
      break;
    }
    ```

    **Step 4: No changes to analyzeExitTriggers function** -- it already passes ExitTriggerConfig objects through to evaluateTrigger. The `entryCost` field will be set on each config object by the tool handler (Plan 03), and evaluateTrigger will read it. The pure engine needs no structural change.
