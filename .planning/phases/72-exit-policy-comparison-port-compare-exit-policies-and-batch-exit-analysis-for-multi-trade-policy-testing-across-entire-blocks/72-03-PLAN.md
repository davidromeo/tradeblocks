Implementation steps:

1. **Query trades from DuckDB** (per D-06, D-07, D-08):

   Use the ROW_NUMBER approach with a `rowid` tiebreaker for deterministic ordering:

   ```sql
   WITH numbered AS (
     SELECT *, ROW_NUMBER() OVER (ORDER BY date_opened, rowid) - 1 AS trade_idx
     FROM trades.trade_data
     WHERE block_id = '{block_id}'
   )
   SELECT trade_idx, legs, pl, date_opened, date_closed, ticker, num_contracts, strategy
   FROM numbered
   WHERE 1=1
   -- AND strategy ILIKE '%{strategy}%'
   -- AND date_opened >= '{from}' AND date_opened <= '{to}'
   -- AND pl >= {min_pl} AND pl <= {max_pl}
   ORDER BY date_opened DESC
   LIMIT {limit}
   ```

   **IMPORTANT:** The `ROW_NUMBER() OVER (ORDER BY date_opened, rowid) - 1` uses `rowid` as a tiebreaker to ensure deterministic trade_idx values for trades sharing the same date_opened. Without the tiebreaker, ROW_NUMBER is non-deterministic for ties.

   Use parameterized escaping: `block_id.replace(/'/g, "''")` (same pattern as replay.ts line 169).

2. **For each trade, call handleReplayTrade** with `block_id` and the trade's row index:

   Call `handleReplayTrade({ block_id, trade_index: row.trade_idx, multiplier, format: 'full' }, baseDir, injectedConn)` for each trade.

   **Note:** Always pass `format:'full'` to handleReplayTrade regardless of `params.format` -- the replay needs the full pnlPath for analyzeBatch. `params.format` controls the batch output density (whether perTrade is included in the final result), not the replay resolution.

3. **Build TradeInput array** from replay results:
   For each trade row + its ReplayResult:
   ```typescript
   const tradeInput: TradeInput = {
     tradeIndex: tradeIdx,
     dateOpened: dateOpened,
     actualPnl: pl,  // from tradelog
     pnlPath: replayResult.pnlPath,
     legs: replayResult.legs,
   };
   ```

4. **Build BatchExitConfig:**
   ```typescript
   const config: BatchExitConfig = {
     candidatePolicy: params.candidate_policy as ExitTriggerConfig[],
     legGroups: params.leg_groups?.map(g => ({
       label: g.label,
       legIndices: g.leg_indices,
       triggers: g.triggers as ExitTriggerConfig[],
     })),
     baselineMode: params.baseline_mode,
     format: params.format,
   };
   ```

5. **Call `analyzeBatch(tradeInputs, config)`** to get BatchExitResult.

6. **Profile context (per D-16):** If strategy is specified, attempt to load profile:
   ```typescript
   if (params.strategy) {
     try {
       const conn = injectedConn ?? await getConnection(baseDir);
       const profile = await getProfile(conn, params.block_id, params.strategy);
       if (profile) {
         result.profileContext = {
           structureType: profile.structureType,
           exitRules: profile.exitRules.map(r => r.description ?? `${r.field} ${r.operator} ${r.value}`),
         };
       }
     } catch { /* Profile is informational context, not critical */ }
   }
   ```

7. **Error handling:** Wrap in try/catch. If handleReplayTrade fails for a trade (e.g., no option bars available), skip that trade and continue. Track skipped count. Include in summary: "Analyzed N trades (M skipped due to replay errors)."

8. **Return BatchExitResult.**

**Registration function:**
```typescript
export function registerBatchExitAnalysisTools(
  server: McpServer,
  baseDir: string,
): void {
  server.registerTool(
    "batch_exit_analysis",
    {
      description:
        "Analyze how a candidate exit policy would perform across multiple trades in a block. " +
        "Replays each matching trade, evaluates exit triggers against the minute-level P&L path, " +
        "and returns aggregate statistics (win rate, Sharpe, profit factor, drawdown) comparable " +
        "to get_statistics. Includes per-trigger attribution showing which triggers drive outcomes. " +
        "First run fetches option bars from Massive (requires MASSIVE_API_KEY); subsequent runs " +
        "use cached data from market.intraday. Use with strategy profiles to iterate on exit rules.",
      inputSchema: batchExitAnalysisSchema,
    },
    async (params) => {
      try {
        const result = await handleBatchExitAnalysis(params, baseDir);
        return createToolOutput(result.summary, result);
      } catch (error) {
        return {
          content: [{
            type: "text" as const,
            text: `Error in batch exit analysis: ${(error as Error).message}`,
          }],
          isError: true,
        };
      }
    }
  );
}
```
