**Test cases (10 minimum):**

1. **`analyzeBatch with empty trades returns zero stats`**
   - Input: `trades = []`, candidatePolicy with profitTarget threshold 500
   - Assert: aggregate.totalTrades === 0, aggregate.winRate === 0, aggregate.totalPnl === 0
   - Assert: triggerAttribution is empty array
   - Assert: perTrade is empty array

2. **`analyzeBatch with 3 winning trades (profitTarget fires) returns correct stats`**
   - Build 3 TradeInput objects each with a pnlPath that climbs to 600 (above 500 threshold)
   - candidatePolicy: `[{ type: 'profitTarget', threshold: 500 }]`
   - baselineMode: 'actual', actualPnl for each trade = 300 (less than candidate)
   - Assert: aggregate.winRate === 1.0
   - Assert: aggregate.totalTrades === 3
   - Assert: aggregate.totalPnl close to 1500 (3 * 500 since trigger fires at >= 500)
   - Assert: aggregate.profitFactor === Infinity (no losses)

3. **`analyzeBatch with mixed wins and losses returns correct profit factor`**
   - 2 winning trades (pnl climbs to 400, profitTarget at 300) + 1 losing trade (pnl drops to -200, stopLoss fires at 200)
   - candidatePolicy: `[{ type: 'profitTarget', threshold: 300 }, { type: 'stopLoss', threshold: 200 }]`
   - Assert: winRate close to 0.667
   - Assert: profitFactor = sum(wins) / abs(sum(losses)) = 600 / 200 = 3.0

4. **`baselineMode actual computes delta as candidatePnl - actualPnl`**
   - Trade with candidatePnl = 500 (profitTarget fires), actualPnl = 200
   - baselineMode: 'actual'
   - Assert: pnlDelta === 300 (candidate was better)

5. **`baselineMode holdToEnd uses last path point as baseline`**
   - Path: [0, 500, 300, 100]. profitTarget fires at 500.
   - baselineMode: 'holdToEnd'
   - candidatePnl = 500 (trigger fired), baselinePnl = 100 (last point)
   - Assert: pnlDelta === 400

6. **`per-trigger attribution counts first-fire correctly`**
   - 3 trades: trade 1 has profitTarget fire first, trade 2 has stopLoss fire first, trade 3 has profitTarget fire first
   - Assert: attribution for profitTarget has count=2
   - Assert: attribution for stopLoss has count=1
   - Assert: attribution sorted by count descending (profitTarget first)

7. **`computeAggregateStats computes max sequential drawdown`**
   - TradeExitResults with candidatePnls: [100, -50, -80, 200, -30]
   - Equity curve: [100, 50, -30, 170, 140]
   - Max drawdown = peak 100 to trough -30 = 130
   - Assert: maxDrawdown === 130

8. **`computeAggregateStats computes Sharpe ratio`**
   - TradeExitResults with candidatePnls: [100, 200, 150, 100, 200]
   - mean = 150, sample stddev = sqrt(sum((x-150)^2) / 4) = ~47.43
   - sharpe = 150 / 47.43 = ~3.162
   - Assert: sharpeRatio is close to 3.162 (within 0.01)

9. **`computeAggregateStats returns null sharpeRatio with < 2 trades`**
   - Single trade result
   - Assert: sharpeRatio === null

10. **`trade where no trigger fires uses noTrigger attribution`**
    - Path that never reaches profitTarget or stopLoss: [0, 50, 100, 80]
    - candidatePolicy: `[{ type: 'profitTarget', threshold: 500 }]` (never reached)
    - Assert: trade result has triggerFired === 'noTrigger'
    - Assert: candidatePnl === 80 (last path point, since no trigger fired = hold to end of replay)
    - Assert: triggerAttribution includes entry with trigger='noTrigger', count=1

11. **`format summary omits perTrade, format full includes it`**
    - Same input, run twice with format='summary' and format='full'
    - Assert: summary result has perTrade.length === 0
    - Assert: full result has perTrade.length > 0

12. **`computeAggregateStats streak counting`**
    - TradeExitResults ordered: [win, win, win, loss, loss, win]
    - Assert: maxWinStreak === 3, maxLossStreak === 2
