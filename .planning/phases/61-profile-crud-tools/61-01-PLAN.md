**1. profile_strategy** (PROF-01)
- Zod input schema:
  - `blockId` (z.string(), required) — aliased to `block_id` in description for Claude convenience but camelCase for TS
  - `strategyName` (z.string(), required)
  - `structureType` (z.string(), required) — description should list known values: iron_condor, calendar_spread, double_calendar, vertical_spread, butterfly, reverse_iron_condor, short_put_spread, etc.
  - `greeksBias` (z.string(), required) — description lists: theta_positive, vega_negative, delta_neutral, etc.
  - `thesis` (z.string().default(""))
  - `legs` (z.array(z.object({ type: z.string(), strike: z.string(), expiry: z.string(), quantity: z.number() })).default([]))
  - `entryFilters` (z.array(z.object({ field: z.string(), operator: z.string(), value: z.union([z.string(), z.number(), z.array(z.union([z.string(), z.number()]))]) , description: z.string().optional() })).default([]))
  - `exitRules` (z.array(z.object({ type: z.string(), trigger: z.string(), description: z.string().optional() })).default([]))
  - `expectedRegimes` (z.array(z.string()).default([]))
  - `keyMetrics` (z.object({ expectedWinRate: z.number().optional(), targetPremium: z.number().optional(), maxLoss: z.number().optional(), profitTarget: z.number().optional() }).passthrough().default({}))
- Handler: Use `withSyncedBlock` middleware. Inside handler: `upgradeToReadWrite(baseDir)`, call `upsertProfile(conn, profileData)`, `downgradeToReadOnly(baseDir)` in finally block. Return the full stored profile via `createToolOutput("Profile saved: {strategyName} for block {blockId}", { profile: storedProfile })`.
- Note: `withSyncedBlock` syncs then downgrades to RO. The handler needs to re-upgrade to RW for the write. Follow the market-imports pattern: `upgradeToReadWrite` + `try/finally { downgradeToReadOnly }` INSIDE the handler.

**2. get_strategy_profile** (PROF-02)
- Zod input: `blockId` (string, required), `strategyName` (string, required)
- Handler: Use `withSyncedBlock`. Call `getProfile(conn, blockId, strategyName)`. If null, return error response "No profile found for strategy '{strategyName}' in block '{blockId}'". Otherwise return full profile via `createToolOutput`.

**3. list_profiles** (PROF-03)
- Zod input: `blockId` (string, optional)
- Handler: When blockId provided, use `withSyncedBlock`. When omitted, NO sync needed — just query directly. Call `listProfiles(conn, blockId)`. Format output as summary rows: for each profile extract `{ blockId, strategyName, structureType, greeksBias, updatedAt }`. Return via `createToolOutput("Found {N} profile(s)", { count, profiles: summaryRows })`.
- Implementation note for optional blockId: Since `withSyncedBlock` requires `blockId` in input, handle the two cases separately. When blockId is present, wrap with `withSyncedBlock`. When absent, call `listProfiles(conn)` directly without sync middleware (matching the CONTEXT.md decision: "When omitted, list across all blocks without sync").

**4. delete_profile**
- Zod input: `blockId` (string, required), `strategyName` (string, required)
- Handler: Use `withSyncedBlock`. `upgradeToReadWrite(baseDir)`, call `deleteProfile(conn, blockId, strategyName)`, `downgradeToReadOnly(baseDir)` in finally. If deleted (true), return `createToolOutput("Deleted profile: {strategyName} from block {blockId}", { deleted: true })`. If not found (false), return `createToolOutput("No profile found for strategy '{strategyName}' in block '{blockId}' — nothing to delete", { deleted: false })`. NOT an error — idempotent per CONTEXT.md decision.

**Connection access:** Use `getConnection(baseDir)` from `../../db/connection.js` to get the DuckDB connection inside handlers (same pattern as all existing tools).

**Wire registration:**
- In `packages/mcp-server/src/index.ts`: Add `import { registerProfileTools } from "./tools/profiles.js"` and call `registerProfileTools(server, resolvedDir)` after the existing registerGuideTools line.
- In `packages/mcp-server/src/cli-handler.ts`: Add same import and call `registerProfileTools(mockServer, resolvedDir)` after registerGuideTools line.
