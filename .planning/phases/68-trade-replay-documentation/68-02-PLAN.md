    ```typescript
    const replayTradeSchema = z.object({
      // Mode A: Hypothetical / explicit legs (per D-03)
      legs: z.array(z.object({
        ticker: z.string().describe("Underlying ticker, e.g., 'SPY', 'SPX'"),
        strike: z.number().describe("Strike price"),
        type: z.enum(["C", "P"]).describe("Call or Put"),
        expiry: z.string().describe("Expiration date YYYY-MM-DD"),
        quantity: z.number().describe("Positive = long, negative = short"),
        entry_price: z.number().describe("Per-contract entry price (premium paid/received)"),
      })).optional().describe("Explicit leg definitions for hypothetical replay"),

      // Mode B: Tradelog replay (per D-01, D-02)
      block_id: z.string().optional().describe("Block ID to load trade from"),
      trade_index: z.number().optional().describe("0-based index of trade in block's tradelog (ordered by date_opened)"),

      // Common fields
      open_date: z.string().optional().describe("Trade open date YYYY-MM-DD (required for hypothetical mode, auto-resolved for tradelog mode)"),
      close_date: z.string().optional().describe("Trade close date YYYY-MM-DD (required for hypothetical, auto-resolved for tradelog)"),
      multiplier: z.number().default(100).describe("Contract multiplier (default 100 for standard options)"),
    });
    ```

    **Handler logic:**

    1. **Input validation:**
       - If `legs` provided → hypothetical mode, require `open_date` and `close_date`
       - If `block_id` + `trade_index` provided → tradelog mode
       - Neither → error: "Provide either legs[] for hypothetical mode or block_id + trade_index for tradelog mode"

    2. **Tradelog mode** (per D-02, RPL-06):
       - Query DuckDB: `SELECT legs, premium, date_opened, date_closed, ticker, num_contracts FROM trades.trade_data WHERE block_id = ? ORDER BY date_opened LIMIT 1 OFFSET ?`
       - Use `getConnection(baseDir)` — this is a READ-ONLY operation (no upgradeToReadWrite needed)
       - Parse the `legs` column with `parseLegsString()`
       - Use trade's `ticker` as root for OCC construction
       - Use trade's `date_opened` / `date_closed` for date range
       - Use trade's `premium` as entry price (divided by num_contracts for per-contract)
       - Build OCC tickers with `buildOccTicker(root, date_closed_as_expiry, parsedLeg.type, parsedLeg.strike)`
         - Note: expiry = date_closed is an approximation. For exact expiry, the legs string or trade record would need it. Use date_closed as best available.
       - Construct ReplayLeg[] with quantity from parsedLeg.quantity * num_contracts

    3. **Hypothetical mode** (per D-03):
       - Map input legs directly to ReplayLeg[] using buildOccTicker()
       - Use provided open_date, close_date, entry_price, quantity

    4. **Fetch minute bars** (per D-13, RPL-01):
       - For each leg, call `fetchBars({ ticker: occTicker, from: open_date, to: close_date, timespan: "minute", assetClass: "option" })`
       - Collect bars into `barsByLeg: MassiveBarRow[][]`

    5. **Compute P&L path** (per D-06, D-07, D-08):
       - Call `computeStrategyPnlPath(replayLegs, barsByLeg)`
       - Call `computeReplayMfeMae(pnlPath)`

    6. **Return** via `createToolOutput(summary, replayResult)`:
       - Summary: "Replayed {N}-leg strategy from {open_date} to {close_date}: {totalPnl} P&L, MFE={mfe}, MAE={mae}, {pnlPath.length} minute bars"

    **Export:** `export { registerReplayTools, handleReplayTrade, replayTradeSchema }` (handler + schema exported for test access)
