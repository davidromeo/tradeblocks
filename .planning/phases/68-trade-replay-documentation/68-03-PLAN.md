    ```
    # Architecture

    ## Overview
    Options trading analytics platform: Next.js 15 web dashboard + MCP server for AI analysis.

    ## Data Flow
    1. Import: CSV upload (web) or MCP tools (import_market_csv, import_from_massive)
    2. Storage: IndexedDB (client-side trades), DuckDB (server-side analytics + market data)
    3. Enrichment: Automatic pipeline adds derived fields (RSI, ATR, VIX regime, IVR/IVP)
    4. Analysis: 50+ MCP tools for statistics, simulation, and strategy profiling

    ## DuckDB Schema
    ### analytics.duckdb (trades database)
    - trades.trade_data — individual trade records
    - trades.reporting_data — reported/live trades for backtest comparison
    - trades._sync_metadata — block sync state

    ### market.duckdb (market database)
    - market.daily — daily OHLCV + enriched fields (RSI, ATR, EMA, etc.)
    - market.context — VIX term structure (VIX, VIX9D, VIX3M) + IVR/IVP
    - market.intraday — minute/hourly bars
    - market._sync_metadata — import tracking

    ## Enrichment Tiers
    - Tier 1: Technical indicators on market.daily (RSI_14, ATR_14, EMA_9, SMA_20, etc.)
    - Tier 2: VIX context fields on market.context (IVR, IVP, Vol_Regime, Term_Structure)

    ## Key Patterns
    - Block-based organization (each strategy = one block)
    - Lookahead-free analytics (close-derived fields use LAG())
    - Eastern Time throughout (all dates are US market dates)
    - MCP tool pattern: Zod schema → handler → RW lifecycle → createToolOutput

    ## Date Handling
    - Trade dates from CSVs: local midnight Date objects, compare with string YYYY-MM-DD
    - Market data timestamps: Unix milliseconds → ET conversion via toLocaleDateString("en-CA", { timeZone: "America/New_York" })
    - Never mix the two approaches (see CLAUDE.md for detailed rules)

    ## Strategy Profiles
    - Stored in DuckDB profiles schema
    - Capture structure_type, legs, entry_filters, exit_rules, expected_regimes
    - Enable structure-aware analysis (structure_fit, filter validation, portfolio mapping)
    ```

    Structure:
    ```
    # Market Data Guide

    TradeBlocks supports two paths for importing market data: CSV files and the Massive.com API.

    ## CSV Import
    ### import_market_csv
    - Requires: file_path, ticker, target_table, column_mapping
    - target_table options: "daily", "context", "intraday"
    - Example daily: { "Date": "date", "Open": "open", "High": "high", "Low": "low", "Close": "close" }
    - Example intraday from TradingView: { "time": "date", "open": "open", "high": "high", "low": "low", "close": "close" }
    - Supports dry_run=true for validation

    ### import_from_database
    - Import from external DuckDB file via SQL query
    - Tables referenced with 'ext_import_source' alias

    ## Massive.com API Import
    ### Setup
    1. Get API key from massive.com
    2. Set environment variable: `export MASSIVE_API_KEY=your_key_here`
    3. Or add to .env file / Claude Desktop config

    ### import_from_massive
    - Parameters: ticker, from, to, target_table
    - target_table="daily": Fetches OHLCV bars → market.daily
    - target_table="context": Auto-fetches VIX, VIX9D, VIX3M → market.context
    - target_table="intraday": Fetches minute bars with configurable timespan
    - skip_enrichment=true to skip auto-enrichment after import

    ### Ticker Formats
    | Type | Example | API Format | Storage Format |
    |------|---------|------------|----------------|
    | Stock | SPY | SPY | SPY |
    | Index | VIX | I:VIX | VIX |
    | Option | SPY 01/17/25 470C | O:SPY250117C00470000 | SPY250117C00470000 |

    The API automatically adds/removes I: and O: prefixes — use plain tickers in tool calls.

    ### OCC Option Ticker Format
    Format: {ROOT}{YYMMDD}{C|P}{STRIKE*1000 padded to 8 digits}
    - SPY Jan 17 2025 470 Call → SPY250117C00470000
    - SPX Dec 19 2025 4500 Put → SPX251219P04500000

    ## Trade Replay
    ### replay_trade
    Replay historical trades using Massive.com option minute bars.

    **Hypothetical mode**: Provide explicit legs with strikes, expiry, entry prices
    **Tradelog mode**: Provide block_id + trade_index to replay from existing trade data

    Returns minute-by-minute P&L path with MFE (max favorable excursion) and MAE (max adverse excursion).

    ## Enrichment Pipeline
    After daily imports, enrichment runs automatically:
    - Tier 1 (market.daily): RSI_14, ATR_14, EMA_9, SMA_20, Realized_Vol, Gap_Pct, etc.
    - Tier 2 (market.context): VIX_IVR, VIX_IVP, VIX9D_IVR, VIX9D_IVP, VIX3M_IVR, VIX3M_IVP, Vol_Regime, Term_Structure

    Use skip_enrichment=true on import or run enrich_market_data separately.

    ## Target Tables Reference
    | Table | Schema | Primary Use |
    |-------|--------|-------------|
    | market.daily | date, ticker, OHLCV + enriched | Daily price data + technical indicators |
    | market.context | date + VIX columns + enriched | VIX term structure + regime classification |
    | market.intraday | date, time, ticker, OHLCV | Minute/hourly bar data |
    ```
